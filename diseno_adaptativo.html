Diseño Adaptativo: Tipografía e Imágenes Responsivas

En esta página muestro cómo se usan unidades relativas, tipografía responsiva e imágenes adaptativas para mejorar la experiencia del usuario en diferentes dispositivos.

Unidades de Medida CSS

Demostración práctica de diferentes unidades de medida y sus casos de uso:

Título en REM

Este título usa 2rem. Las unidades rem son relativas al elemento raíz (html), lo que las hace predecibles y escalables. Ideal para títulos porque escalan uniformemente con el zoom del navegador.

Uso: Títulos, elementos que necesitan escalado consistente

Subtítulo en EM

Este texto usa 1.1em. Las unidades em son relativas al elemento padre, por lo que escalan proporcionalmente. Si el padre es 16px, este texto será 17.6px.

Uso: Texto que debe escalar con su contexto, componentes anidados

Bloque con altura VH

Este contenedor usa 30vh (30% del viewport height).

Uso: Heroes, secciones de pantalla completa

Ancho en VW

Usa 80vw (80% del viewport width).

Uso: Elementos fluidos, contenedores adaptativos

Tamaño VMIN

Font-size de 4vmin. Se basa en la dimensión más pequeña del viewport.

Uso: Texto legible en cualquier orientación

Tamaño VMAX

Font-size de 3vmax. Se basa en la dimensión más grande del viewport.

Uso: Elementos decorativos prominentes

Imágenes Responsivas

Ejemplo 1: Imagen con srcset y sizes

Técnica: srcset con descriptores de ancho (w) y atributo sizes
Funcionamiento: 400w para móviles, 800w para tablets, 1200w para desktop
Beneficio: Optimiza la carga según el dispositivo

Esta imagen cambia de composición según el dispositivo:

Ejemplos de Código

Unidades Responsivas

/* REM - relativo al root */
.titulo { font-size: 2rem; }

/* EM - relativo al padre */
.subtitulo { font-size: 1.5em; }

/* VH - porcentaje de altura */
.hero { height: 100vh; }

/* VW - porcentaje de ancho */
.container { width: 80vw; }

/* VMIN y VMAX */
.text { font-size: 4vmin; }
.big { font-size: 3vmax; }

Tipografía Responsiva

/* clamp() - automático */
.text {
    font-size: clamp(1rem, 2.5vw, 1.4rem);
    /* mínimo | preferido | máximo */
}

/* calc() - combinado */
.text {
    font-size: calc(1rem + 0.5vw);
}

Imágenes Responsivas

/* srcset con sizes */
<img src="image.jpg"
     srcset="image-400.jpg 400w,
             image-800.jpg 800w,
             image-1200.jpg 1200w"
     sizes="(max-width: 480px) 100vw,
            (max-width: 1024px) 80vw,
            60vw"
     alt="Responsive image">

/* picture para dirección de arte */
<picture>
    <source media="(max-width: 480px)"
            srcset="vertical.jpg">
    <source media="(max-width: 1024px)"
            srcset="square.jpg">
    <img src="horizontal.jpg" alt="Art direction">
</picture>

Conceptos Clave

Unidades Relativas vs Absolutas

  • Absolutas (px): Fijas, no escalan con el viewport
  • Relativas (rem, em, %, vw, vh): Se adaptan automáticamente
  • rem: Basado en el tamaño de fuente raíz (html)
  • em: Basado en el tamaño de fuente del padre
  • vw/vh: Porcentaje del viewport (1vw = 1% del ancho)
  • vmin/vmax: Basado en la dimensión menor/mayor del viewport

Tipografía Responsiva

  • clamp(): Define tamaño mínimo, preferido y máximo automáticamente
  • calc(): Combina unidades (ej: calc(1rem + 0.5vw))
  • @font-face: Importa fuentes personalizadas
  • Google Fonts: Biblioteca de fuentes web gratuitas
  • Line-height: Mejora la legibilidad (1.6-1.8 recomendado)

Imágenes Adaptativas

  • srcset: Diferentes resoluciones para diferentes pantallas
  • sizes: Define cuándo usar cada imagen del srcset
  • <picture>: Dirección de arte (diferentes composiciones)
  • object-fit: Controla cómo se ajusta la imagen (cover, contain)
  • Lazy loading: Carga diferida con loading="lazy"

Mobile First

El enfoque Mobile First consiste en diseñar primero para dispositivos móviles y luego escalar hacia arriba. Esto garantiza que la experiencia sea óptima en todos los dispositivos, priorizando el contenido esencial y mejorando el rendimiento.

Resumen del Diseño Adaptativo

Ventajas

  • Mejor experiencia de usuario en todos los dispositivos
  • Mayor accesibilidad y usabilidad
  • Mejor posicionamiento SEO
  • Reducción de costos de mantenimiento
  • Carga optimizada según dispositivo

Mejores Prácticas

  • Usar unidades relativas (rem, em, %, vw, vh)
  • Implementar tipografía fluida con clamp()
  • Optimizar imágenes con srcset y picture
  • Probar en dispositivos reales
  • Aplicar enfoque Mobile First
